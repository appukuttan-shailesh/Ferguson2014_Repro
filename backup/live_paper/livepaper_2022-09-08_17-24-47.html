Abstract:
Neuronal models are representations of real neurons that attempt to mimic certain aspects of physiologically realistic behavior. It is difficult to accommodate all the cellular and sub-cellular phenomena within these models owing to both a lack of complete understanding of these mechanisms, as well as increased computational complexity. Point-neuron models are simplified representations that ignore the spatial extent of neurons while focusing on simulating the experimentally observed spiking activity. Owing to their lowered complexity, these models are particularly suited for network-level investigations where the computational load rises exponentially. The development of biologically-constrained point-neuron models is therefore vital to studying neuronal networks. We attempted to reproduce one such work by Ferguson et al. (2014). They developed models capturing the frequency-current profiles of both strongly and weakly adapting CA1 pyramidal cells using the Brian simulator. As part of our study, we reproduced the model on different simulators such as Brian2 and NEURON. The reproductions were attempted independent of the available Brian implementation, relying solely based on the published article. The different implementations were then quantitatively validated, against the original model, using the EBRAINS Model Validation Framework, to evaluate how well they mirror the original model. Additional tests were developed and packaged into a test suite that helped further characterize and compare various aspects of these models, beyond the scope of the original study. All resources employed and developed in our study have been shared publicly via the EBRAINS Live Papers, thereby enabling replication and verification of our findings.
Resources

All resources that were created and/or employed in our study, have been publicly shared here through this Live Paper. In addition, all the models and tests discussed in this study have been registered on the EBRAINS Model Validation Framework, thereby providing access to additional relevant metadata. Every validation result, along with data files and figures produced as part of the evaluation, are linked to the models and the tests, and publicly available online. These measures greatly help in satisfying the Findable (F), Accessible (A), and Reusable (R) aspects of the FAIR principles for sharing scientific data. Our GitHub repository also provides access to the files associated with this study.

  • exposure eFELunit Test Suite

    A test suite, named eFELunit, was developed in Python as a distributable package, containing several validation tests in addition to those described in the original study. This allows for a more comprehensive evaluation of the various aspects of the different implementations of the model.

    Figure shows the SciUnit based workflow of running validation tests. The capability acts as an interface between the models and tests, and defines the functionalities demanded from the models. The solid arrows indicate the functions (in red) that are to be implemented by our models; these functions represent more granular functionalities. The function (in blue) combines these more basic functions, to provide a more comprehensive workflow, and can be directly utilized (shown by dashed arrows) by the tests to evaluate the model.

    You can pip install the package as follows:

      pip install eFELunit

    (note: use version >= 2.0.1)

    Example usage:

      data = [{"i_inj": "50 pA", "value": "60.99 mV"}, {...}] # specify reference data
  from eFELunit.tests import eFELfeatureTest
  test = eFELfeatureTest(data, feature="AP1_amp", force_run=True)
  from Brian2.generic_Pyr_model_Brian2 import CA1_Pyr_Brian2_Template
  model = CA1_Pyr_Brian2_Template(type='strong')
  test.judge(model, deep_error=True)
